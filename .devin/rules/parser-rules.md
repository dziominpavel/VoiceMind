# ReminderParser

## Контракт
- Чистая функция / класс **без Android SDK**.
- Зависит от передаваемого `Instant now` (или `Clock`) для тестов.
- Выход: `ParseResult` — `fireAt`, `body`, `confidence`, `warnings`, `matchedTimeSpan`, `rawPhrase`.

## Разделение фразы (candidate-based engine)
1. **Найти ВСЕ матчи** regex (`findAll`) → создать `DateCandidate` / `TimeCandidate` со `score`.
2. `resolveBestDate` / `resolveBestTime` выбирают по **max score** (при равенстве — раньше в строке).
3. Часть дня (утром/вечером) **не переопределяет** точное время `HH:mm`.
4. `body` = исходная строка минус spans выбранных кандидатов, trim, нормализация пробелов.
5. Убрать вводные в начале: «напомни», «напоминание», «нужно», «не забудь».

## Поддерживаемые конструкции (ru-RU, MVP)
| Тип | Примеры |
|-----|---------|
| Относительные дни | сегодня, завтра, послезавтра, через N дней |
| Дни недели | в понедельник, во вторник — **ближайший**; если время прошло — +7 дней |
| Время | 9:00, 21:30, в 9 часов, в 9 часов 30 минут |
| Периоды | утром → 09:00, днём → 13:00, вечером → 19:00, ночью → 22:00 |
| AM/PM-стиль | в 9 вечера → 21:00, в 9 утра → 09:00 |
| Спец | полночь / полдень / 12 ночи / 12 дня |
| Относительное | через N минут/часов, через полчаса (+30), через полтора часа (+90) |
| Абсолютная дата | 25 мая, 25 мая 2026, dd.MM — год: если в прошлом → **следующий год** |

## Confidence
| Условие | Значение |
|---------|----------|
| Дата + время явно | 0.92 |
| Относительный день + «утром» | 0.75 |
| «через час» / «через полчаса» | 0.85 |
| «в девять» без уточнения | 0.5 + warning |
| Время не найдено | 0 + `NO_TIME_FOUND` |

При `confidence < 0.7` — показывать предупреждение жёлтым на ConfirmScreen.

## Warnings
- `TIME_AMBIGUOUS` — «в девять» без утра/вечера.
- `DATE_MISSING_YEAR` — «3 апреля» подставлен год.
- `NO_TIME_FOUND` — время не найдено.
- `BODY_EMPTY` — body пуст после удаления времени.
- `PAST_TIME_ADJUSTED` — «сегодня в 8 утра» уже прошло → предложить завтра.

## Прошедшее время
- Если `fireAt < now` и была «сегодня» утром → warning + предложить завтра.
- Иначе — не сохранять автоматически, пользователь правит на ConfirmScreen.

## Тесты
- **55+** фиксированных фраз с замороженным `Instant now`.
- Edge: полночь, полдень, «в 12 ночи», «через полчаса», «завтра» на 23:59.
- Новый паттерн → сначала **test-case**, потом реализация.
