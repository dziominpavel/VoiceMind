## Purpose

Мастер надёжности, запрос критичных разрешений, тестовое напоминание и индикаторы на главном экране.

## Requirements

### Requirement: Мастер надёжности при первом запуске
При первом запуске приложение MUST показывать мастер надёжности, который последовательно объясняет и запрашивает критичные для срабатывания разрешения: уведомления (`POST_NOTIFICATIONS`, API 33+), точные будильники (`SCHEDULE_EXACT_ALARM`, API 31+) и исключение из оптимизации батареи. Мастер MUST показывать только те шаги, которые актуальны для текущей версии Android и ещё не выполнены. После прохождения или пропуска мастер MUST NOT показываться повторно автоматически.

#### Scenario: Первый запуск без разрешений
- **WHEN** пользователь впервые открывает приложение и критичные разрешения не выданы
- **THEN** показывается мастер надёжности с шагами по невыданным разрешениям
- **AND** каждый шаг содержит объяснение, зачем разрешение нужно для срабатывания

#### Scenario: Мастер не повторяется
- **WHEN** пользователь прошёл или пропустил мастер
- **THEN** при следующих запусках мастер автоматически не показывается
- **AND** флаг `onboarding_completed` сохранён в настройках

#### Scenario: Только актуальные шаги
- **WHEN** устройство на API ниже 33
- **THEN** шаг `POST_NOTIFICATIONS` в мастере не показывается

### Requirement: Тестовое напоминание
Приложение MUST предоставлять действие «проверить срабатывание», которое создаёт реальное напоминание через стандартный путь планирования (`ReminderRepository.insertAndSchedule`) на момент `now + ~60 секунд`.

#### Scenario: Проверка доставки
- **WHEN** пользователь нажимает «Проверить, что работает»
- **THEN** создаётся напоминание с `fireAt = now + ~60 секунд` и понятным текстом
- **AND** оно планируется тем же путём, что и обычные напоминания
- **AND** при срабатывании пользователь видит доставку выбранным режимом

### Requirement: Индикатор надёжности на главном экране
Главный экран MUST показывать постоянный индикатор-предупреждение, если не выданы критичные для срабатывания разрешения (уведомления, точные будильники или исключение из оптимизации батареи). Индикатор MUST давать переход к исправлению и MUST скрываться, когда все критичные условия выполнены.

#### Scenario: Чего-то не хватает
- **WHEN** на главном экране отсутствует хотя бы одно критичное разрешение
- **THEN** показывается баннер «напоминания могут не сработать» с кнопкой «Исправить»
- **AND** кнопка открывает мастер надёжности

#### Scenario: Всё выдано
- **WHEN** все критичные разрешения выданы
- **THEN** баннер на главном экране не показывается

#### Scenario: Состояние обновляется при возврате
- **WHEN** пользователь изменил разрешение в системных настройках и вернулся в приложение
- **THEN** индикатор надёжности пересчитывается при возобновлении экрана

### Requirement: Повторный вызов мастера из настроек
Пользователь MUST иметь возможность повторно открыть мастер надёжности из экрана настроек.

#### Scenario: Запуск мастера из настроек
- **WHEN** пользователь выбирает «Проверка надёжности» в настройках
- **THEN** открывается мастер надёжности независимо от флага `onboarding_completed`
