## Purpose

Экраны приложения VoiceMind, навигация, UI-компоненты NeoWave Voice и взаимодействие пользователя.
## Requirements
### Requirement: Тёмная тема по умолчанию
Приложение MUST использовать тёмную тему NeoWave Voice по умолчанию. Светлая тема — фаза 4+.

#### Scenario: Первый запуск
- **WHEN** пользователь открывает приложение впервые
- **THEN** активна тёмная тема с палитрой NeoWave Voice
- **AND** фон BackgroundPrimary #040608, поверхности SurfaceElevated #182231
- **AND** используется tonal elevation вместо теней для разделения уровней

### Requirement: Акцентный цвет бирюза
UI MUST использовать акцентный цвет бирюза #2EC4B6 для ключевых элементов. К палитре добавлены производные токены: AccentGlow #53F2E1, AccentSoft #1B6E65, AccentContainer #103B37.

#### Scenario: Главный экран
- **WHEN** пользователь на главном экране
- **THEN** кнопка микрофона — бирюзовая с радиальным свечением AccentGlow при нажатии
- **AND** ближайшее напоминание выделяется бирюзовым отсчётом времени

#### Scenario: Просроченное напоминание
- **WHEN** напоминание просрочено
- **THEN** левая акцентная полоса карточки — TimeCritical #FF5A5F, пульсирующая

### Requirement: Главный экран — ближайшее напоминание
Главный экран MUST показывать ближайшее предстоящее напоминание в Hero-карточке (220dp, скругление 36dp) с радиальным свечением акцента, или empty state с waveform-иллюстрацией.

#### Scenario: Есть предстоящие напоминания
- **WHEN** на главном экране есть SCHEDULED напоминания
- **THEN** отображается Hero Card ближайшего напоминания
- **AND** время отображается шрифтом displayXL (48sp) с tabular nums
- **AND** под временем — CountdownLabel (14sp, accentPrimary): "через N мин/ч/дн"
- **AND** ниже Hero Card — до 3 карточек предстоящих напоминаний
- **AND** каждая карточка предпросмотра показывает время (titleMedium) и метку даты (labelSmall, textMuted): «сегодня», «завтра» или «d MMM»

#### Scenario: Нет напоминаний
- **WHEN** нет предстоящих напоминаний
- **THEN** показывается empty state с иконкой waveform и текстом «Скажите первое напоминание»
- **AND** подсказка: «Например: «Завтра в девять позвонить маме»»

### Requirement: Экран подтверждения
После парсинга MUST открываться экран подтверждения с редактируемыми полями, Quick Chips для быстрого выбора даты/времени и Confidence Indicator. Режим доставки на этом экране НЕ выбирается — применяется глобальный из настроек.

#### Scenario: Подтверждение голосового напоминания
- **WHEN** парсер вернул результат с confidence >= 0.7
- **THEN** открывается ConfirmScreen с датой, временем и текстом
- **AND** `DeliveryModeGrid` на экране НЕ отображается
- **AND** поля даты/времени кликабельны (pickers)
- **AND** под полем времени — ряд Quick Chips: «Сегодня», «Завтра», «Через час», «Вечером», «Выбрать»
- **AND** выбор chip сразу обновляет fireAtMillis

#### Scenario: Низкий confidence
- **WHEN** парсер вернул результат с confidence < 0.7
- **THEN** показывается Confidence Indicator (цветная точка + текст) вместо полноразмерного жёлтого блока
- **AND** цвет точки: warning для 40–70%, error для <40%
- **AND** пользователь может отредактировать дату, время и текст
- **AND** `DeliveryModeGrid` на экране НЕ отображается

### Requirement: Список напоминаний
Экран списка MUST иметь две вкладки: Предстоящие и История. Напоминания группируются по дате с sticky-заголовками.

#### Scenario: Вкладка Предстоящие
- **WHEN** пользователь открывает вкладку Предстоящие
- **THEN** показываются SCHEDULED напоминания, отсортированные по fireAt ASC
- **AND** группировка по дате: «Сегодня», «Завтра», «На неделе», «Позже»
- **AND** каждая группа имеет sticky-заголовок
- **AND** просроченные имеют пульсирующую красную левую полосу и красный tint фона (6%)
- **AND** срочные (<24ч) имеют жёлтую левую полосу
- **AND** критические (<1ч) имеют красную левую полосу
- **AND** левая цветная полоса (accent bar) имеет фиксированную ширину и не перекрывает содержимое карточки
- **AND** в правой части карточки показывается иконка текущего глобального режима доставки из настроек (одинаковая для всех карточек)
- **AND** чекбокса «выполнено» в списке НЕТ
- **AND** LazyColumn имеет contentPadding снизу 16dp, чтобы последняя карточка не прилипала к BottomAppBar

#### Scenario: Вкладка История
- **WHEN** пользователь открывает вкладку История
- **THEN** показываются FIRED, DISMISSED, CANCELLED, отсортированные fireAt DESC
- **AND** выполненные имеют зачёркнутый текст и зелёную левую полосу

#### Scenario: Swipe-to-Reveal для отмены
- **WHEN** пользователь свайпает карточку предстоящего напоминания влево
- **THEN** карточка сдвигается влево и залипает на фиксированную ширину reveal-панели (80dp)
- **AND** reveal-панель имеет фон errorContainer только в полосе действия справа, не на всю ширину строки
- **AND** скруглённые углы карточки MUST NOT просвечивать красную подложку
- **AND** reveal-панель показывает красную иконку удаления, центрированную в полосе 80dp
- **AND** иконка удаления кликабельна
- **AND** иконка режима доставки скрыта, пока reveal-панель открыта
- **AND** иконка удаления MUST NOT наезжать на иконку режима доставки
- **AND** при тапе по иконке удаления напоминание отменяется (статус CANCELLED)
- **AND** при тапе по сдвинутой карточке она возвращается в исходное положение
- **AND** при открытии reveal-панели на одной карточке остальные reveal-панели закрываются

#### Scenario: Время и дата в карточке
- **WHEN** карточка предстоящего напоминания отображается
- **THEN** левая колонка времени (72dp) показывает время в формате HH:mm (titleMedium)
- **AND** под временем показывается относительная дата в compact-формате: «сегодня», «завтра», «послезавтра» или «d MMM» (labelSmall, textMuted)
- **AND** полная дата со временем НЕ показывается в колонке времени
- **AND** между текстом напоминания и иконкой режима доставки есть отступ 16dp (Spacing.md)
- **AND** иконка режима доставки не наезжает на текст напоминания

### Requirement: Экран деталей напоминания
Экран деталей MUST позволять просматривать информацию о напоминании, редактировать, отменять, откладывать и отмечать как выполненное.

#### Scenario: Детали запланированного напоминания
- **WHEN** пользователь открывает детали предстоящего (SCHEDULED) напоминания
- **THEN** показываются статус, дата/время, текст напоминания
- **AND** в BottomAppBar доступны кнопки: «Отменить», «Отложить», «Готово»
- **AND** тап по «Готово» меняет статус на DONE и отменяет alarm
- **AND** тап по «Отменить» меняет статус на CANCELLED и отменяет alarm
- **AND** тап по «Отложить» открывает выбор времени откладывания

### Requirement: ConfirmReminderScreen — IME
Экран подтверждения MUST корректно обрабатывать открытие клавиатуры.

#### Scenario: Редактирование body с открытой клавиатурой
- **WHEN** пользователь редактирует поле body на ConfirmReminderScreen
- **AND** открывается программная клавиатура
- **THEN** Scaffold сдвигается вверх через imePadding
- **AND** кнопка "Сохранить" в BottomAppBar остаётся видимой

### Requirement: SnackbarHost — единственный экземпляр
Приложение MUST иметь ровно один SnackbarHost на SnackbarHostState.

#### Scenario: Отображение ошибки
- **WHEN** происходит ошибка и ViewModel эмитит сообщение
- **THEN** показывается один Snackbar внутри Scaffold
- **AND** дублирующего SnackbarHost снаружи Scaffold нет

### Requirement: Ручное создание напоминания
Пользователь MUST иметь возможность создать напоминание вручную через форму. Режим доставки на форме НЕ выбирается — применяется глобальный из настроек.

#### Scenario: Ручной ввод
- **WHEN** пользователь нажимает "Ввести текстом"
- **THEN** открывается ManualReminderScreen с DatePicker, TimePicker, полем body
- **AND** дефолтное время — завтра 9:00
- **AND** форма чистая, без блока предложений
- **AND** `DeliveryModeGrid` на экране НЕ отображается

### Requirement: Экран настроек
Настройки MUST содержать выбор режима доставки по умолчанию как radio-группу из 5 компактных карточек. Глобальный toggle вибрации отсутствует.

#### Scenario: Настройки — выбор режима доставки
- **WHEN** пользователь открывает настройки
- **THEN** секции сгруппированы в карточки (surfaceElevated, radius 28dp)
- **AND** в карточке "Режим по умолчанию" доступны 5 компактных radio-карточек с иконками в порядке от громкого к тихому:
  - «Будильник» — ALARM с вибрацией (ringtone + full-screen + вибрация), default
  - «Будильник без вибрации» — ALARM без вибрации (ringtone + full-screen)
  - «Вибрация» — VIBRATE (только вибро)
  - «Уведомление» — NOTIFICATION (системный звук + короткая вибрация)
  - «Тихий» — SILENT (ничего)
- **AND** ровно один режим может быть выбран одновременно
- **AND** карточки имеют уменьшенный padding (10dp vertical, 12dp horizontal) чтобы 5 пунктов влезали на экран
- **AND** карточка «Будильник» или «Будильник без вибрации» открывает дополнительную карточку с выбором мелодии и громкостью
- **AND** toggle "Подтверждать перед созданием" доступен в отдельной карточке «Поведение»
- **AND** глобального Switch вибрации под radio-группой НЕТ

#### Scenario: Переключение между режимами
- **WHEN** пользователь выбирает «Будильник» или «Будильник без вибрации»
- **THEN** появляется карточка «Будильник» с настройками мелодии и громкости
- **AND** при переключении на другой режим эта карточка скрывается

#### Scenario: Карточка «О приложении»
- **WHEN** пользователь прокручивает настройки до конца
- **THEN** в карточке «О приложении» показывается строка «Версия X.Y.Z» (без типа сборки)
- **AND** ниже — строка «Разработчик Дёмин Павел"
- **AND** не показывается тестовая подсказка

### Requirement: NeoMicButton с glow и waveform
Кнопка микрофона MUST иметь радиальное свечение, пульсирующее кольцо и имитируемую волновую форму.

#### Scenario: Состояние Listening
- **WHEN** пользователь нажимает кнопку микрофона
- **THEN** кнопка увеличивается до 124dp (scale 1.08)
- **AND** появляется радиальное свечение AccentGlow с alpha 0.35
- **AND** пульсирует внешнее кольцо (scale 1.0 → 1.45, alpha 0.5 → 0, 1200ms loop)
- **AND** под кнопкой появляются 5 анимированных баров waveform
- **AND** генерируется haptic Medium

#### Scenario: Состояние Idle
- **WHEN** микрофон не активен
- **THEN** кнопка 112dp, статичная, без waveform
- **AND** при нажатии масштабируется до 0.92 (tween 120ms)

#### Scenario: Reduced motion
- **WHEN** в системе включён reduced-motion
- **THEN** waveform — статичные бары средней высоты
- **AND** glow-pulse отключён
- **AND** scale-анимации заменены на fade

### Requirement: ReminderCard с urgency states
Карточка напоминания MUST иметь левую акцентную полосу 4dp и 8 визуальных состояний.

#### Scenario: Нормальное состояние
- **WHEN** напоминание SCHEDULED, >24ч
- **THEN** левая полоса OutlineStrong (3dp)
- **AND** фон Surface

#### Scenario: Срочное состояние
- **WHEN** напоминание SCHEDULED, <=24ч и >1ч
- **THEN** левая полоса TimeWarning (4dp)
- **AND** фон Surface с янтарным tint 4%

#### Scenario: Критическое состояние
- **WHEN** напоминание SCHEDULED, <=1ч
- **THEN** левая полоса TimeCritical (4dp)
- **AND** фон Surface с красным tint 4%

#### Scenario: Просроченное состояние
- **WHEN** напоминание SCHEDULED, fireAt < now
- **THEN** левая полоса TimeCritical (4dp), пульсирующая alpha 0.5→1 (1000ms loop)
- **AND** фон Surface с красным tint 6%
- **AND** текст метки: «Просрочено»

#### Scenario: Выполненное состояние
- **WHEN** напоминание DISMISSED
- **THEN** левая полоса Success (3dp)
- **AND** текст body зачёркнут, alpha 0.7

### Requirement: Alarm Fired экран
Экран срабатывания будильника MUST быть полноэкранным, OLED-чёрным, с крупным временем и минимумом отвлекающих элементов.

#### Scenario: Отображение срабатывания
- **WHEN** срабатывает alarm в режиме ALARM
- **THEN** открывается Activity с фоном BackgroundPrimary #040608
- **AND** время отображается шрифтом displayXL (48sp)
- **AND** body напоминания — headlineSmall (24sp)
- **AND** кнопка «Выполнено» — 72dp высота, pill, accentPrimary, со свечением
- **AND** за кнопкой — breathing glow (alpha 0.05→0.15, 3000ms loop)

### Requirement: Haptic feedback
Все ключевые действия пользователя MUST сопровождаться тактильной обратной связью.

#### Scenario: Сохранение напоминания
- **WHEN** пользователь нажимает «Сохранить»
- **THEN** генерируется haptic Success

#### Scenario: Удаление
- **WHEN** пользователь свайпает карточку для удаления
- **THEN** генерируется haptic Heavy

#### Scenario: Начало прослушивания
- **WHEN** пользователь нажимает кнопку микрофона
- **THEN** генерируется haptic Medium

### Requirement: Snackbar
Система уведомлений об ошибках/успехе MUST использовать кастомный Snackbar, привязанный над BottomNav.

#### Scenario: Ошибка
- **WHEN** происходит ошибка (DB, permission, scheduling)
- **THEN** показывается Snackbar с иконкой error, фон SurfaceElevated, 10s duration
- **AND** позиция: 16dp над BottomNav

#### Scenario: Успех
- **WHEN** напоминание успешно сохранено
- **THEN** показывается Snackbar с иконкой check, duration 4s

### Requirement: Accent bar в карточке списка
Левая цветная полоса (accent bar) в карточке предстоящего напоминания MUST иметь фиксированную ширину и занимать полную высоту карточки, не перекрывая содержимое.

#### Scenario: Корректное отображение accent bar
- **WHEN** карточка предстоящего напоминания отображается
- **THEN** accent bar имеет ширину 4dp
- **AND** accent bar занимает полную высоту карточки
- **AND** accent bar не перекрывает текст напоминания, время или иконку режима доставки
- **AND** содержимое карточки располагается справа от accent bar с отступом `Spacing.md` (16dp)
