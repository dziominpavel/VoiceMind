## Context

- `ReminderRepository.snoozeReminder` уже считает `now + delay` (путь уведомления — корректен).
- `VoiceMindViewModel.snoozeReminder` считал `reminder.fireAt + minutes` → баг для просроченных (исправлено на `now + minutes`).
- `ReminderScheduler.schedule` и `ReminderRepository.updateAndSchedule` пропускают планирование, если `fireAt <= now` — поэтому ошибочный snooze «терялся» молча.
- Детали: пресеты 10/30/60 (`ReminderDetailScreen`); ALARM: фиксированные 10 (`AlarmActivity`).

## Goals / Non-Goals

**Goals:**
- Единая семантика snooze «от now» на всех путях.
- Гибкие пресеты: 5/15/60 мин + «завтра утром».
- Корректный snooze для просроченных напоминаний.

**Non-Goals:**
- Произвольный date-time picker для snooze (можно позже; «завтра утром» закрывает основную потребность).
- Настраиваемое «утреннее время» в UI (берём дефолт 09:00; вынос в настройки — отдельный инкремент).
- Изменение быстрого действия в самом уведомлении (остаётся +10 мин).

## Decisions

### 1. Snooze всегда от `now`

- Все вызовы snooze вычисляют `newFireAt = System.currentTimeMillis() + delta`.
- Это совпадает со спекой и с путём уведомления; устраняет «потерянные» просроченные напоминания.

### 2. «Завтра утром»

- Helper: ближайшее наступление времени `09:00` строго в будущем (если сейчас до 09:00 — сегодня 09:00; иначе завтра 09:00). Считать в локальной TZ устройства (`Calendar`/`LocalDateTime`).
- Передаём в существующий путь snooze как абсолютный `fireAt`, а не «минуты» → потребуется перегрузка/вариант метода snooze, принимающий абсолютный момент, либо вычисление минут до цели. Решение: добавить во ViewModel `snoozeUntil(id, fireAtMillis)` рядом с `snoozeReminder(id, minutes)`, оба идут через `updateAndSchedule`.

### 3. Пресеты UI

- `ReminderDetailScreen`: заменить список на 5/15/60 + «завтра утром».
- ALARM-экран: добавить компактный выбор (или те же пресеты), сохранив быстрый дефолт.
- **Альтернатива (отклонена):** полноценный picker на ALARM-экране — избыточно для моментального действия спросонья.

## Risks / Trade-offs

- **[«Завтра утром» в абсолютных millis vs минуты]** → вводим `snoozeUntil`, не ломая существующий `snoozeReminder(minutes)`.
- **[Часовой пояс/переход на летнее время]** → используем системный календарь локальной TZ, не ручную арифметику суток.
- **[ALARM-экран усложнится]** → держим минимальный набор кнопок, дефолт остаётся быстрым.

## Migration Plan

1. (Сделано) Исправить `VoiceMindViewModel.snoozeReminder` на `now + minutes`.
2. Добавить helper «завтра утром» + `snoozeUntil`.
3. Обновить пресеты в `ReminderDetailScreen` и ALARM-экране.
4. Строки + регресс-проверка просроченного snooze.
- **Rollback:** изменения локальны (ViewModel + два экрана + helper).
