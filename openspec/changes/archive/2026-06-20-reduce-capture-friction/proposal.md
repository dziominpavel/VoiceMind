## Why

Ценность голосовой напоминалки — захват мысли за секунды: «вспомнил → сказал → готово». Сейчас путь длиннее, чем нужно:

- чтобы продиктовать, надо открыть приложение и нажать микрофон (нет быстрых точек входа из системы);
- есть виджет с микрофоном, но нет Quick Settings tile и app-shortcut (long-press по иконке);
- шаг подтверждения (`confirmBeforeSchedule`) по умолчанию включён — для быстрых очевидных фраз это лишний тап.

Каждый лишний шаг снижает вероятность, что пользователь вообще запишет напоминание.

## What Changes

- **Quick Settings tile:** плитка в шторке, запускающая немедленный голосовой захват.
- **App shortcut:** статический ярлык «Новое напоминание (голос)» по long-press иконки приложения.
- **Быстрый захват без лишних шагов:** маршрут «из системы → сразу слушаем», открывающий приложение прямо в режиме записи.
- **Пересмотр поведения подтверждения:** дать «быстрое сохранение» для уверенного результата парсинга (high confidence), оставив подтверждение для неоднозначных. Точное поведение по умолчанию — открытый вопрос (см. design.md), без принудительной смены без согласования.

## Capabilities

### New Capabilities

- `quick-capture`: системные точки входа для мгновенного голосового захвата (QS tile, app shortcut, deep-link «сразу слушать») и правила быстрого сохранения.

### Modified Capabilities

(возможна правка дефолта подтверждения в `notification-delivery`/настройках — только после согласования; в этом change по умолчанию поведение не меняется принудительно.)

## Impact

- **Манифест:** `TileService` для QS tile; `shortcuts.xml` + `meta-data` для статических shortcut'ов.
- **Код:** новый `TileService`, обработка shortcut/tile-intent в `MainActivity` (запуск слушания), переиспользование существующего пути голосового ввода и виджет-экшенов (`WidgetActions.ACTION_START_VOICE`).
- **Без изменений** схемы Room и парсера.
- **Совместимость:** QS tile — API 24+; shortcuts — API 25+; на старых API просто отсутствуют, базовый поток не страдает.
