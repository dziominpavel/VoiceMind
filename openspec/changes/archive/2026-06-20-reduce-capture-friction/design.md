## Context

- Уже есть голосовой захват в приложении и виджет с микрофоном; `WidgetActions.ACTION_START_VOICE` запускает слушание, `MainActivity.handleIntent` его обрабатывает (`fallbackToSystemSpeech`).
- Нет QS tile и app-shortcut.
- `confirmBeforeSchedule` по умолчанию включён; `ParseResult` несёт `confidence`.

## Goals / Non-Goals

**Goals:**
- Дать 1–2 системные точки входа для мгновенного голосового захвата без ручного открытия и тапа по микрофону.
- Переиспользовать существующий путь голосового ввода (не дублировать логику).

**Non-Goals:**
- Принудительно менять дефолт подтверждения без согласования с владельцем продукта.
- Always-on / hotword-активация (за рамками; дорого по батарее и приватности).
- Замена виджета (он остаётся).

## Decisions

### 1. Переиспользование существующего маршрута захвата

- QS tile и shortcut шлют в `MainActivity` тот же интент-маршрут, что и виджет (`ACTION_START_VOICE`), чтобы был один источник логики старта слушания.
- `TileService.onClick` → `startActivityAndCollapse` с интентом запуска слушания.

### 2. App shortcut — статический

- `res/xml/shortcuts.xml` со статическим shortcut «Новое напоминание (голос)», deep-link в `MainActivity` с тем же action.
- Статический проще динамического и достаточно для одной фиксированной точки входа.

### 3. Быстрое сохранение по confidence (предложение, не дефолт по умолчанию)

- Когда `confirmBeforeSchedule` выключен ИЛИ результат парсинга high-confidence и время в будущем — сохранять сразу с ненавязчивым undo (Snackbar «Запланировано · Отменить»).
- Неоднозначные/без времени — вести в подтверждение/ручной ввод.
- **Открытый вопрос:** делать ли это поведением по умолчанию. Без согласования — оставляем текущий дефолт, но реализуем механику «быстрое сохранение + undo» под существующий флаг.
- **Альтернатива (отклонена):** всегда сохранять без подтверждения — рискует молча запланировать неверно распознанное.

## Risks / Trade-offs

- **[Tile/shortcut запускают слушание при заблокированном экране]** → стартуем Activity штатно; распознавание требует разблокировки/разрешений — обрабатываем как обычный голосовой поток.
- **[Дубли логики старта]** → строго переиспользуем `ACTION_START_VOICE`-маршрут.
- **[Смена дефолта подтверждения раздражает]** → не меняем без согласования; вводим только undo-механику.

## Migration Plan

1. Добавить `shortcuts.xml` + `TileService` в манифест и ресурсы.
2. Обработать tile/shortcut-интенты в `MainActivity` (переиспользовать стартовый маршрут).
3. (Опционально, по согласованию) включить «быстрое сохранение + undo».
4. Строки, иконки tile/shortcut, ручная проверка на API 24/25+.
- **Rollback:** удаление tile-сервиса и shortcuts.xml; базовый поток не зависит от них.
