## MODIFIED Requirements

### Requirement: Список напоминаний
Экран списка MUST иметь две вкладки: Предстоящие и История. Напоминания группируются по дате с sticky-заголовками. Подпись «просрочено» MUST появляться только у `PENDING` с `fireAt < now` во вкладке Предстоящие; в Истории MUST NOT показываться «просрочено».

#### Scenario: Вкладка Предстоящие
- **WHEN** пользователь открывает вкладку Предстоящие
- **THEN** показываются PENDING напоминания, отсортированные по fireAt ASC
- **AND** группировка по дате: «Сегодня», «Завтра», «На неделе», «Позже»
- **AND** каждая группа имеет sticky-заголовок
- **AND** просроченные (`PENDING`, `fireAt < now`) имеют пульсирующую красную левую полосу и красный tint фона (6%), наложенный на непрозрачный базовый фон
- **AND** срочные (<24ч) имеют жёлтую левую полосу
- **AND** критические (<1ч) имеют красную левую полосу
- **AND** фон каждой карточки MUST быть полностью непрозрачным (alpha = 1.0), независимо от urgency-состояния
- **AND** красная reveal-панель удаления MUST NOT быть видна в покое (без свайпа)
- **AND** левая цветная полоса (accent bar) имеет фиксированную ширину и не перекрывает содержимое карточки
- **AND** в правой части карточки показывается иконка текущего глобального режима доставки из настроек (одинаковая для всех карточек)
- **AND** чекбокса «выполнено» в списке НЕТ
- **AND** LazyColumn имеет contentPadding снизу 16dp, чтобы последняя карточка не прилипала к BottomAppBar

#### Scenario: Вкладка История
- **WHEN** пользователь открывает вкладку История
- **THEN** показываются TRIGGERED, DONE, CANCELLED, отсортированные fireAt DESC
- **AND** под временем показывается статус текстом («Выполнено» / «Отменено» / «Без ответа») и короткая дата («сегодня» / «вчера» / «d MMM»)
- **AND** подпись «просрочено» MUST NOT отображаться
- **AND** DONE: зелёная левая полоса (3dp), body зачёркнут, muted
- **AND** CANCELLED: серая левая полоса (2dp), body muted, без зачёркивания
- **AND** TRIGGERED: оранжевая левая полоса TimeWarning (3dp), body обычный

#### Scenario: Swipe-to-Reveal для отмены
- **WHEN** пользователь свайпает карточку предстоящего напоминания влево
- **THEN** карточка сдвигается влево и залипает на фиксированную ширину reveal-панели (80dp)
- **AND** reveal-панель имеет фон errorContainer только в полосе действия справа, не на всю ширину строки
- **AND** reveal-панель и иконка удаления отображаются только при сдвиге карточки (offset < 0)
- **AND** скруглённые углы карточки MUST NOT просвечивать красную подложку
- **AND** reveal-панель показывает красную иконку удаления, центрированную в полосе 80dp
- **AND** иконка удаления кликабельна
- **AND** иконка режима доставки скрыта, пока reveal-панель открыта
- **AND** иконка удаления MUST NOT наезжать на иконку режима доставки
- **AND** при тапе по иконке удаления напоминание отменяется (статус CANCELLED)
- **AND** при тапе по сдвинутой карточке она возвращается в исходное положение
- **AND** при открытии reveal-панели на одной карточке остальные reveal-панели закрываются

#### Scenario: Время и дата в карточке предстоящего
- **WHEN** карточка предстоящего напоминания отображается
- **THEN** левая колонка времени (72dp) показывает время в формате HH:mm (titleMedium)
- **AND** под временем для будущего fireAt показывается compact-дата: «сегодня», «завтра», «послезавтра» или «d MMM» (labelSmall, textMuted)
- **AND** под временем для просроченного PENDING показывается «просрочено»
- **AND** полная дата со временем НЕ показывается в колонке времени
- **AND** между текстом напоминания и иконкой режима доставки есть отступ 16dp (Spacing.md)
- **AND** иконка режима доставки не наезжает на текст напоминания

### Requirement: Экран деталей напоминания
Экран деталей MUST позволять просматривать информацию о напоминании, редактировать, отменять, откладывать и отмечать как выполненное. Откладывание MUST предлагать гибкие пресеты: 5, 15 и 60 минут, а также «завтра утром». Любой пресет MUST вычислять новое время от текущего момента. Кнопка завершения MUST называться «Выполнено».

#### Scenario: Детали запланированного напоминания
- **WHEN** пользователь открывает детали предстоящего (PENDING) напоминания
- **THEN** показываются статус, дата/время, текст напоминания
- **AND** в BottomAppBar доступны кнопки: «Отменить», «Отложить», «Выполнено»
- **AND** тап по «Выполнено» меняет статус на DONE и отменяет alarm
- **AND** тап по «Отменить» меняет статус на CANCELLED и отменяет alarm
- **AND** тап по «Отложить» открывает выбор пресета откладывания

#### Scenario: Пресеты откладывания
- **WHEN** пользователь открывает выбор откладывания
- **THEN** доступны варианты 5 / 15 / 60 минут и «завтра утром»
- **AND** выбор минутного пресета планирует `now + N минут`
- **AND** выбор «завтра утром» планирует ближайшее наступление утреннего времени (по умолчанию 09:00) в будущем

#### Scenario: Лейблы статусов на деталях
- **WHEN** открыты детали напоминания
- **THEN** бейдж статуса показывает: PENDING → «Ожидает», TRIGGERED → «Без ответа», DONE → «Выполнено», CANCELLED → «Отменено»

### Requirement: ReminderCard с urgency states
Карточка предстоящего напоминания MUST визуально отражать срочность через левую полосу и фон. Urgency-шкала применяется только к PENDING во вкладке Предстоящие. Terminal-статусы Истории кодируются отдельно (см. «Список напоминаний»).

#### Scenario: Нормальное состояние
- **WHEN** напоминание PENDING, >24ч до fireAt
- **THEN** левая полоса OutlineStrong (3dp)
- **AND** фон Surface непрозрачный

#### Scenario: Срочное состояние
- **WHEN** напоминание PENDING, <=24ч и >1ч до fireAt
- **THEN** левая полоса TimeWarning (4dp)
- **AND** фон Surface с янтарным tint 4%, наложенным непрозрачно на базовый Surface

#### Scenario: Критическое состояние
- **WHEN** напоминание PENDING, <=1ч до fireAt и fireAt >= now
- **THEN** левая полоса TimeCritical (4dp)
- **AND** фон Surface с красным tint 4%, наложенным непрозрачно на базовый Surface

#### Scenario: Просроченное состояние
- **WHEN** напоминание PENDING, fireAt < now
- **THEN** левая полоса TimeCritical (4dp), пульсирующая alpha 0.5→1 (1000ms loop)
- **AND** фон Surface с красным tint 6%, наложенным непрозрачно на базовый Surface
- **AND** текст метки: «просрочено»

#### Scenario: Выполненное в Истории
- **WHEN** напоминание DONE отображается во вкладке История
- **THEN** левая полоса Success/TimeSafe (3dp)
- **AND** текст body зачёркнут, muted
- **AND** под временем показан лейбл «Выполнено» и короткая дата (не «просрочено»)
