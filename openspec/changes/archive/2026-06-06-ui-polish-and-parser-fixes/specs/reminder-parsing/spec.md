## MODIFIED Requirements

### Requirement: Приоритет маркера части дня над точным временем (вечер/ночь)
Когда в фразе присутствует и точное время (HH:mm), и маркер части дня «вечера/вечером/ночи/ночью», парсер ДОЛЖЕН учитывать маркер части дня и корректировать час.

#### Scenario: Точное время с маркером вечера
- **WHEN** пользователь говорит «сегодня в 8:00 вечера спать»
- **THEN** кандидат TIME_HOURS_PART (span «в 8:00 вечера», score 100) выбирается над TIME_COLON (span «8:00», score 90)
- **AND** парсер вычисляет час как 20 (8 + 12), потому что маркер — «вечера"
- **AND** fireAt = сегодня 20:00
- **AND** body = «спать»

#### Scenario: Точное время с маркером ночи
- **WHEN** пользователь говорит «сегодня в 2:00 ночи подняться"
- **THEN** TIME_HOURS_PART (span «в 2:00 ночи», score 100) выбирается над TIME_COLON (span «2:00», score 90)
- **AND** fireAt = сегодня 02:00 (ночь — hour без изменения)
- **AND** body = «подняться»

#### Scenario: Маркер дня/утра не переопределяет точное время >= 12
- **WHEN** пользователь говорит «завтра в 18:30 днём обед»
- **THEN** TIME_HOURS_PART (span «в 18:30 днём», score 80) проигрывает TIME_COLON (span «18:30", score 90)
- **AND** fireAt = завтра 18:30 (exact time wins для дня/утра)
- **AND** body = «обед»

#### Scenario: Маркер дня с голым часом < 12
- **WHEN** пользователь говорит «в 3 дня обед»
- **THEN** TIME_HOURS_PART (span «в 3 дня», score 80) выбирается
- **AND** парсер вычисляет час как 15 (3 + 12)
- **AND** fireAt = сегодня 15:00
- **AND** body = «обед»

## IMPLEMENTATION NOTE
- TIME_HOURS_PART score = 100 только для маркеров «вечера/вечером/ночи/ночью"
- TIME_HOURS_PART score = 80 для маркеров «утра/утром/дня/днём/днем"
- TIME_COLON score = 90 (ниже вечерных HOURS_PART, выше дневных)
- При hour >= 12 для маркеров «дня/вечера» +12 не применяется (защита от wrap-around)
