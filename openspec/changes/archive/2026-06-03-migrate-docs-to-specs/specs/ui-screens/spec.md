## ADDED Requirements

### Requirement: Тёмная тема по умолчанию
Приложение ДОЛЖНО использовать тёмную тему CLEAR BELL по умолчанию. Светлая тема — фаза 4+.

#### Scenario: Первый запуск
- **WHEN** пользователь открывает приложение впервые
- **THEN** активна тёмная тема с палитрой CLEAR BELL
- **AND** фон Midnight #0A0C10, поверхности Slate #141820

### Requirement: Акцентный цвет бирюза
UI ДОЛЖЕН использовать акцентный цвет бирюза #2EC4B6 для ключевых элементов.

#### Scenario: Главный экран
- **WHEN** пользователь на главном экране
- **THEN** кнопка микрофона — бирюзовая
- **AND** ближайшее напоминание выделяется бирюзовым

### Requirement: Главный экран — ближайшее напоминание
Главный экран ДОЛЖЕН показывать ближайшее предстоящее напоминание или empty state.

#### Scenario: Есть предстоящие напоминания
- **WHEN** на главном экране есть SCHEDULED напоминания
- **THEN** отображается одна карточка ближайшего
- **AND** время отображается крупным шрифтом с tabular nums

#### Scenario: Нет напоминаний
- **WHEN** нет предстоящих напоминаний
- **THEN** показывается empty state с призывом создать

### Requirement: Экран подтверждения
После парсинга ДОЛЖЕН открываться экран подтверждения с редактируемыми полями.

#### Scenario: Подтверждение голосового напоминания
- **WHEN** парсер вернул результат с confidence >= 0.7
- **THEN** открывается ConfirmScreen с датой, временем, текстом и режимом
- **AND** поля даты/времени кликабельны (pickers)

#### Scenario: Низкий confidence
- **WHEN** парсер вернул результат с confidence < 0.7
- **THEN** на ConfirmScreen показывается жёлтое предупреждение
- **AND** пользователь может отредактировать все поля

### Requirement: Список напоминаний
Экран списка ДОЛЖЕН иметь две вкладки: Предстоящие и История.

#### Scenario: Вкладка Предстоящие
- **WHEN** пользователь открывает вкладку Предстоящие
- **THEN** показываются SCHEDULED напоминания, отсортированные по fireAt ASC
- **AND** просроченные имеют subtle error tint

#### Scenario: Вкладка История
- **WHEN** пользователь открывает вкладку История
- **THEN** показываются FIRED, DISMISSED, CANCELLED, отсортированные fireAt DESC

### Requirement: Ручное создание напоминания
Пользователь ДОЛЖЕН иметь возможность создать напоминание вручную через форму.

#### Scenario: Ручной ввод
- **WHEN** пользователь нажимает "Ввести текстом"
- **THEN** открывается ManualReminderScreen с DatePicker, TimePicker, полем body
- **AND** дефолтное время — завтра 9:00

### Requirement: Экран настроек
Настройки ДОЛЖНЫ содержать выбор режима по умолчанию и toggle confirmBeforeSchedule.

#### Scenario: Настройки
- **WHEN** пользователь открывает настройки
- **THEN** доступен radio-group режимов с пояснением
- **AND** toggle "Подтверждать перед созданием"

### Requirement: DeliveryModePicker
UI ДОЛЖЕН содержать компонент выбора режима оповещения с иконками.

#### Scenario: Выбор режима
- **WHEN** пользователь создаёт или редактирует напоминание
- **THEN** отображается DeliveryModePicker с иконками колокол, вибро, без звука
- **AND** выбранный режим подсвечивается
