## REMOVED Requirements

### Requirement: Запрос исключения из оптимизации батареи
**Reason**: Для MVP напоминания планируются через `AlarmManager.setExactAndAllowWhileIdle`, который уже обходит Doze для одиночных срабатываний; recurring перепланируется внутри `ReminderAlarmReceiver` через тот же путь. Между срабатываниями приложение не ведёт фоновой работы, поэтому исключение из оптимизации батареи не приносит измеримой пользы. Permission `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` является flagged в Google Play и требует обоснования при публикации. Эмпирически без exemption напоминания срабатывали стабильно.
**Migration**: Никакой миграции данных не требуется — состояние battery exemption не хранилось в DataStore, а проверялось в рантайме через `PowerManager.isIgnoringBatteryOptimizations()`. Пользователям, уже прошедшим onboarding, wizard автоматически не показывается. Шаг battery optimization просто исчезает из wizard для новых пользователей; индикатор надёжности на главном экране перестаёт учитывать состояние battery optimization и показывается только при отсутствии `POST_NOTIFICATIONS` или `SCHEDULE_EXACT_ALARM`. Если в будущем потребуется борьба с OEM-киллерами (Xiaomi/Huawei), это будет отдельная capability с OEM-специфичными инструкциями, не возвращающая flagged permission.
