## Context

- `ReminderParser` — императивный парсер на Kotlin, обрабатывающий русские фразы; находит дату, время, текст; возвращает `ParseResult` с `confidence`, `warnings`, `fireAt`.
- Текущие паттерны: «завтра», «через N дней/часов/минут», «в HH:MM», конкретные даты, названия месяцев.
- Нет: слов-чисел, полудниц, мягких интервалов, диапазонов.

## Goals / Non-Goals

**Goals:**
- Распознавать слова-числа для часов, минут, дней.
- Распознавать «утром/в обед/днём/вечером/ночью» как временные привязки.
- Распознавать «к/около/примерно» как приблизительное время (confidence < 1.0).
- Распознавать диапазоны «с X до Y» (в MVP — начало диапазона).

**Non-Goals:**
- Полная поддержка всех падежей русского языка (только nom/acc/prepositional для чисел).
- Морфологический анализатор (пока ручной маппинг; если станет сложно — рассмотреть библиотеку позже).
- Произвольные именные даты («через две недели после вторника») — за рамками MVP.
- Встроенный справочник праздников (в MVP placeholder/warning).

## Decisions

### 1. Слова-числа

- Таблица маппинга 0–59 (для минут) и 0–23 (для часов), а также интервальные («один/два/три...часа/дней/минут»).
- Триггеры: «к [число]», «через [число] [единицу]», «на [число] [единицу]».
- Формат: номинатив ("пять") и винительный/предложный ("пяти" для «к пяти»).
- **Альтернатива (отклонена):** морфологический анализатор — тяжело под Android; ручной маппинг покрывает 90% живой речи.

### 2. Полудница и мягкие интервалы

- "утром" → default 09:00, confidence 0.85
- "в обед" → default 13:00, confidence 0.85
- "днём" → default 14:00, confidence 0.80
- "вечером" → default 19:00, confidence 0.80
- "ночью" → default 01:00, confidence 0.70
- Если уже прошло дефолтное время — сдвиг на следующий день (утро→завтра утром).
- Триггеры: "в/во [интервал]", "[интервал]".

### 3. Приблизительное время (к/около/примерно)

- «к пяти» → 17:00 с confidence 0.75 (пользователь готов к ±15 мин).
- «около семи» → 19:00 с confidence 0.70.
- Warning: «время приблизительное».
- **Альтернатива (отклонена):** диапазон confidence — достаточно флага приблизительности.

### 4. Диапазоны

- «с пяти до семи» → парсим как 17:00 (начало), confidence 0.70, warning «указан диапазон».
- В MVP не запрашиваем выбор из диапазона — минимальный вариант.

### 5. Праздники/события

- «на Новый год» → placeholder: warning «уточните конкретную дату», confidence 0.3.
- «на день рождения» → warning «добавьте дату в контакты», confidence 0.2.
- Не пытаемся угадать без данных.

## Risks / Trade-offs

- **[Ручной маппинг чисел разрастается]** → текущий парсер уже императивен, добавление маппинга — локальный инкремент; при 60+ числах — рассмотреть библиотеку.
- **[«В обед» — субъективно]** → confidence < 1.0 + warning; пользователь подтвердит или исправит.
- **[Диапазоны неоднозначны]** → берём начало + warning; минимальный MVP.
- **[Падежи русского языка]** → покрываем nom/acc/prepositional для чисел; остальные — fallback на подтверждение.

## Migration Plan

1. Добавить словари чисел (0–59, 0–23) в парсер.
2. Добавить функции распознавания полудницы, приблизительного времени, диапазонов.
3. Юнит-тесты: по 3–5 фраз на каждую новую категорию.
4. Обновить `ParseWarning` при необходимости.
- **Rollback:** удаление новых паттернов-функций; существующие тесты продолжают проходить.
