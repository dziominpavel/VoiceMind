## ADDED Requirements

### Requirement: Дефолтное утреннее время при явной дате без времени
Если парсер нашёл явную дату, но не нашёл время, система MUST устанавливать `fireAt` на **09:00** (локальный TZ) выбранной даты. Правило едино для всех типов явных дат: относительных (`завтра`, `через неделю`, `через месяц`, `через N дней`), абсолютных (`25 мая`, `9 числа`) и дней недели.

#### Scenario: Завтра без времени
- **WHEN** пользователь говорит «завтра купить молоко»
- **THEN** `fireAt` = завтра 09:00
- **AND** `body` = «купить молоко»
- **AND** warning `NO_TIME_FOUND` присутствует

#### Scenario: Через месяц без времени
- **WHEN** пользователь говорит «через месяц напоминалка четыре»
- **THEN** `fireAt` = та же календарная дата через 1 месяц в 09:00
- **AND** `body` = «напоминалка четыре»
- **AND** warning `NO_TIME_FOUND` присутствует
- **AND** `isVoiceParseSuccessful()` возвращает `true`

### Requirement: Относительные месяцы
Парсер MUST распознавать русские выражения «через месяц» и «через N месяц/месяца/месяцев» и разрешать их в дату `today.plusMonths(N)` (при отсутствии числа N = 1).

#### Scenario: Через месяц
- **WHEN** пользователь говорит «через месяц напоминалка четыре»
- **THEN** дата = сегодня + 1 месяц
- **AND** span «через месяц» удаляется из body
- **AND** `fireAt` = эта дата в 09:00

#### Scenario: Через два месяца
- **WHEN** пользователь говорит «через 2 месяца сдать отчёт»
- **THEN** дата = сегодня + 2 месяца
- **AND** `body` = «сдать отчёт»
- **AND** `fireAt` = эта дата в 09:00

#### Scenario: Через пять месяцев (слово месяцев)
- **WHEN** пользователь говорит «через 5 месяцев позвонить»
- **THEN** дата = сегодня + 5 месяцев
- **AND** `body` = «позвонить»
