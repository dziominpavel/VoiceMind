## MODIFIED Requirements

### Requirement: Экран подтверждения
После парсинга MUST открываться экран подтверждения с редактируемыми полями, Quick Chips для быстрого выбора даты/времени и Confidence Indicator. Режим доставки на этом экране НЕ выбирается — применяется глобальный из настроек.

#### Scenario: Подтверждение голосового напоминания
- **WHEN** парсер вернул результат с confidence >= 0.7
- **THEN** открывается ConfirmScreen с датой, временем и текстом
- **AND** `DeliveryModeGrid` на экране НЕ отображается
- **AND** поля даты/времени кликабельны (pickers)
- **AND** под полем времени — ряд Quick Chips: «Сегодня», «Завтра», «Через час», «Вечером», «Выбрать»
- **AND** выбор chip сразу обновляет fireAtMillis

#### Scenario: Низкий confidence
- **WHEN** парсер вернул результат с confidence < 0.7
- **THEN** показывается Confidence Indicator (цветная точка + текст) вместо полноразмерного жёлтого блока
- **AND** цвет точки: warning для 40–70%, error для <40%
- **AND** пользователь может отредактировать дату, время и текст
- **AND** `DeliveryModeGrid` на экране НЕ отображается

### Requirement: Список напоминаний
Экран списка MUST иметь две вкладки: Предстоящие и История. Напоминания группируются по дате с sticky-заголовками.

#### Scenario: Вкладка Предстоящие
- **WHEN** пользователь открывает вкладку Предстоящие
- **THEN** показываются SCHEDULED напоминания, отсортированные по fireAt ASC
- **AND** группировка по дате: «Сегодня», «Завтра», «На неделе», «Позже»
- **AND** каждая группа имеет sticky-заголовок
- **AND** просроченные имеют пульсирующую красную левую полосу и красный tint фона (6%)
- **AND** срочные (<24ч) имеют жёлтую левую полосу
- **AND** критические (<1ч) имеют красную левую полосу
- **AND** левая цветная полоса (accent bar) имеет фиксированную ширину и не перекрывает содержимое карточки
- **AND** в правой части карточки показывается иконка текущего глобального режима доставки из настроек (одинаковая для всех карточек)
- **AND** чекбокса «выполнено» в списке НЕТ
- **AND** LazyColumn имеет contentPadding снизу 16dp, чтобы последняя карточка не прилипала к BottomAppBar

#### Scenario: Вкладка История
- **WHEN** пользователь открывает вкладку История
- **THEN** показываются FIRED, DISMISSED, CANCELLED, отсортированные fireAt DESC
- **AND** выполненные имеют зачёркнутый текст и зелёную левую полосу

#### Scenario: Swipe-to-Reveal для отмены
- **WHEN** пользователь свайпает карточку предстоящего напоминания влево
- **THEN** карточка сдвигается влево и залипает на фиксированную ширину reveal-панели (80dp)
- **AND** reveal-панель имеет фон errorContainer только в полосе действия справа, не на всю ширину строки
- **AND** скруглённые углы карточки MUST NOT просвечивать красную подложку
- **AND** reveal-панель показывает красную иконку удаления, центрированную в полосе 80dp
- **AND** иконка удаления кликабельна
- **AND** иконка режима доставки скрыта, пока reveal-панель открыта
- **AND** иконка удаления MUST NOT наезжать на иконку режима доставки
- **AND** при тапе по иконке удаления напоминание отменяется (статус CANCELLED)
- **AND** при тапе по сдвинутой карточке она возвращается в исходное положение
- **AND** при открытии reveal-панели на одной карточке остальные reveal-панели закрываются

#### Scenario: Время и дата в карточке
- **WHEN** карточка предстоящего напоминания отображается
- **THEN** левая колонка времени (72dp) показывает время в формате HH:mm (titleMedium)
- **AND** под временем показывается относительная дата в compact-формате: «сегодня», «завтра», «послезавтра» или «d MMM» (labelSmall, textMuted)
- **AND** полная дата со временем НЕ показывается в колонке времени
- **AND** между текстом напоминания и иконкой режима доставки есть отступ 16dp (Spacing.md)
- **AND** иконка режима доставки не наезжает на текст напоминания

### Requirement: Ручное создание напоминания
Пользователь MUST иметь возможность создать напоминание вручную через форму. Режим доставки на форме НЕ выбирается — применяется глобальный из настроек.

#### Scenario: Ручной ввод
- **WHEN** пользователь нажимает "Ввести текстом"
- **THEN** открывается ManualReminderScreen с DatePicker, TimePicker, полем body
- **AND** дефолтное время — завтра 9:00
- **AND** форма чистая, без блока предложений
- **AND** `DeliveryModeGrid` на экране НЕ отображается

## REMOVED Requirements

### Requirement: DeliveryModePicker
**Reason:** Режим доставки стал глобальным из настроек; per-reminder выбор удалён с Confirm/Manual экранов.
**Migration:** Пользователь выбирает режим в настройках; все напоминания используют `defaultDeliveryMode`.
