## Контекст

SettingsScreen сейчас показывает 4 карточки (ALARM, NOTIFICATION, VIBRATE, SILENT) в вертикальном списке + глобальный Switch вибрации. Нужно сделать 5 карточек, убрав Switch.

## Goals / Non-Goals

**Goals:**
- 5 компактных radio-карточек влезают на экран без необходимости scroll.
- Карточки визуально единообразны, понятны без subtitle.
- Глобальный toggle vibrate удалён.

**Non-Goals:**
- Изменение DataStore, ViewModel, ReminderNotifier или другого backend.
- Изменение DeliveryMode enum.
- Изменение ConfirmReminderScreen / ManualReminderScreen.

## Decisions

### 1. Layout: 5 компактных карточек в столбик
**Решение:** Вертикальный список из 5 карточек с уменьшенным padding (12dp внутри вместо 16dp). Title + icon слева, RadioButton справа. Subtitle убран — заменён на более описательные title.

**Альтернатива:** Grid 2×2 + 1 centered внизу. **Отклонено:** на маленьких экранах grid сжимается до нечитаемого размера, вертикальный список предсказуем.

### 2. Названия режимов
**Решение:**
- «Тихий»
- «Уведомление»
- «Вибрация»
- «Будильник» (подразумевает +вибрация)
- «Будильник без вибрации»

**Почему:** явное разделение двух вариантов будильника. Пользователь видит оба и выбирает один.

### 3. Вибрация для NOTIFICATION
**Решение:** NOTIFICATION всегда включает короткую системную вибрацию (как стандартный пуш). Это поведение уже есть в `ReminderNotifier.kt` через `setVibrate(DEFAULT_VIBRATE_PATTERN)` когда `useVibration=true`. Но поскольку глобальный toggle убирается, для NOTIFICATION vibrate становится unconditional.

**Важно:** это меняет поведение NOTIFICATION для пользователей, которые раньше выключили vibrate toggle. Для них теперь пуш всегда вибрирует. Это сознательное упрощение — если не хочешь вибрацию, выбирай SILENT или ALARM без вибро.

## Risks / Trade-offs

| Риск | Компенсация |
|------|-------------|
| Пользователь привык к toggle и не найдёт «без вибро» | 5-я карточка явно названа |
| NOTIFICATION теперь всегда vibrate | Это стандартное поведение пушей в Android; toggle был избыточен |
| Миграция невозможна (нет данных о старом toggle) | UI самодостаточен — пользователь выбирает заново при первом открытии |

## Миграция

Нет миграции данных. UI показывает текущее значение `defaultDeliveryMode` + `useVibration`. При первом открытии после обновления mapping работает корректно:
- ALARM + vibrate ON → «Будильник»
- ALARM + vibrate OFF → «Будильник без вибрации»
- NOTIFICATION → «Уведомление» (vibrate всегда ON)
- VIBRATE → «Вибрация»
- SILENT → «Тихий»
