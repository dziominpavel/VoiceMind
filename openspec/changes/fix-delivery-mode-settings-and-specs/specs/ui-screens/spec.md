## MODIFIED Requirements

### Requirement: Экран настроек
Настройки ДОЛЖНЫ содержать выбор режима доставки по умолчанию как radio-группу из 4 режимов и отдельный toggle вибрации.

#### Scenario: Настройки — выбор режима доставки
- **WHEN** пользователь открывает настройки
- **THEN** секции сгруппированы в карточки (surfaceElevated, radius 28dp)
- **AND** в карточке "Режим по умолчанию" доступна radio-группа из 4 режимов с иконками:
  - «Будильник» — ALARM (ringtone + full-screen)
  - «Уведомление» — NOTIFICATION (стандартный пуш)
  - «Вибрация» — VIBRATE (только вибро)
  - «Тихий» — SILENT (только в шторке)
- **AND** ровно один режим может быть выбран одновременно
- **AND** ниже — отдельный Switch «Вибрация при срабатывании», который влияет на поведение ALARM и NOTIFICATION
- **AND** Switch «Вибрация» скрыт или disabled когда выбран режим VIBRATE (вибрация в этом режиме всегда включена)
- **AND** toggle "Подтверждать перед созданием" доступен в отдельной карточке

#### Scenario: Карточка «О приложении»
- **WHEN** пользователь прокручивает настройки до конца
- **THEN** в карточке «О приложении» показывается строка «Версия X.Y.Z» (без типа сборки)
- **AND** ниже — строка «Разработчик Дёмин Павел"
- **AND** не показывается тестовая подсказка

### Requirement: Ручное создание напоминания
Пользователь ДОЛЖЕН иметь возможность создать напоминание вручную через форму, выбирая режим доставки.

#### Scenario: Ручной ввод с выбором режима
- **WHEN** пользователь нажимает "Ввести текстом"
- **THEN** открывается ManualReminderScreen с DatePicker, TimePicker, полем body
- **AND** дефолтное время — завтра 9:00
- **AND** форма чистая, без блока предложений
- **AND** под полем body отображается `DeliveryModeGrid` с начальным выбором из настроек (`defaultDeliveryMode`)
- **AND** пользователь может изменить режим доставки перед сохранением

### Requirement: Экран подтверждения
После парсинга ДОЛЖЕН открываться экран подтверждения с редактируемыми полями, Quick Chips для быстрого выбора даты/времени, Confidence Indicator, Delivery Mode Grid и начальным режимом из настроек.

#### Scenario: Подтверждение голосового напоминания
- **WHEN** парсер вернул результат
- **THEN** открывается ConfirmScreen с датой, временем, текстом и режимом
- **AND** `DeliveryModeGrid` отображает начальный выбор из `settings.defaultDeliveryMode`
- **AND** поля даты/времени кликабельны (pickers)
- **AND** под полем времени — ряд Quick Chips: «Сегодня», «Завтра», «Через час», «Вечером», «Выбрать»
- **AND** выбор chip сразу обновляет fireAtMillis

#### Scenario: Низкий confidence
- **WHEN** парсер вернул результат с confidence < 0.7
- **THEN** показывается Confidence Indicator (цветная точка + текст)
- **AND** цвет точки: warning для 40–70%, error для <40%
- **AND** пользователь может отредактировать все поля, включая режим доставки
