# Парсинг голосовых напоминаний

> Дата: 2026-06-03 · Статус: **реализовано** (80+ unit-тестов)

## Pipeline

```mermaid
flowchart LR
    A[Голос / текст] --> B[STT]
    B --> C[rawPhrase]
    C --> D[ReminderParser]
    D --> E[ParseResult]
    E --> F[ConfirmScreen]
    F --> G[Reminder в Room]
    G --> H[AlarmManager]
```

1. **STT** — `SpeechRecognizer` (on-device), locale `ru-RU`. Результат: одна строка `rawPhrase`.
2. **ReminderParser** — чистая функция / класс без Android-зависимостей (юнит-тестируемый).
3. **ParseResult** — `fireAt`, `body`, `confidence`, `warnings[]`.
4. Пользователь правит на **ConfirmScreen** → финальные значения уходят в БД.

Сеть **не обязательна**. Опционально (фаза 5): если `confidence < threshold`, один запрос к API «разбери фразу в JSON».

## Контракт ParseResult

```kotlin
data class ParseResult(
    val fireAt: Instant?,           // null если время не найдено
    val body: String,               // текст напоминания
    val confidence: Float,          // 0..1
    val warnings: List<ParseWarning>,
    val matchedTimeSpan: IntRange?, // индексы в rawPhrase для подсветки
    val rawPhrase: String,          // исходная фраза
)

enum class ParseWarning {
    TIME_AMBIGUOUS,      // «в девять» без утра/вечера
    DATE_MISSING_YEAR, // «3 апреля» — подставлен текущий/следующий год
    NO_TIME_FOUND,
    BODY_EMPTY,
    PAST_TIME_ADJUSTED // «в 8 утра» уже прошло → завтра
}
```

## Разделение фразы на «время» и «текст»

**Candidate-based engine** (рефакторинг v2):

1. **Normalize**: strip вводных («напомни», «не забудь», «сделай напоминание», «напомни мне», «чтобы не забыть»), lowercase.
2. **Extract ALL candidates**: каждый regex ищет *все* матчи в строке → создаёт `DateCandidate` / `TimeCandidate` со score.
3. **Resolve**: выбирается лучшая дата (`max score`) и лучшее время (`max score`).
   - Если кандидаты пересекаются — выигрывает более специфичный (например `HH:mm` score 100 vs `утром` score 40).
   - Часть дня (`утром`/`вечером`) **не переопределяет** точное время (`9:00`).
4. **Body** = исходная строка минус spans выбранных кандидатов, trim.

Это позволяет парсить перестановки («позвонить маме завтра в 9:00» → body = «позвонить маме») и разрешать конфликты между паттернами.

Примеры:

| rawPhrase | fireAt (логика) | body |
|-----------|-----------------|------|
| завтра в 9:00 позвонить соседу | завтра 09:00 | позвонить соседу |
| через час купить молоко | now + 1h | купить молоко |
| в понедельник в 18:30 тренировка | ближайший пн 18:30 | тренировка |
| позвонить маме завтра утром | завтра 09:00 (дефолт утро) | позвонить маме |
| 25 мая в 14:00 стоматолог | 25 мая 14:00 тек./след. год | стоматолог |

## Поддерживаемые конструкции (MVP, ru)

### Относительные дни

| Фраза | День |
|-------|------|
| сегодня | calendar today |
| завтра | +1 day |
| послезавтра | +2 days |
| через N дней | +N days |

### Дни недели

`в понедельник`, `во вторник`, … — **ближайший** такой день (если сегодня вторник и сказали «вторник» без «следующий» — сегодня, если время уже прошло — +7 дней).

### «Следующий» день недели

`в следующий понедельник`, `в следующую среду`, `в следующую пятницу` — **+7 дней** от ближайшего такого дня (понедельник следующей недели).

### Выходные

`на выходных` — ближайшая предстоящая суббота.

### Время

| Паттерн | Пример |
|---------|--------|
| HH:mm | 9:00, 09:00, 21:30 |
| HHmm (no separator) | 0051 → 00:51, 2359 → 23:59 |
| H часов | в 9 часов |
| в N часов M минут | в 9 часов 30 минут |
| утром / днём / вечером / ночью | 09:00 / 13:00 / 19:00 / 22:00 (дефолты, настраиваемые) |
| в N часов M минут | в 9 часов 15 минут |
| в N [утра/вечера/дня/ночи] | в 9 вечера → 21:00 |
| полночь / полдень / 12 ночи / 12 дня | 00:00 / 12:00 |
| утром/вечером/днём/ночью в N | утром в 9 → 09:00, вечером в 8 → 20:00 |
| половина N+1 | половина девятого → 8:30 |
| без M N | без пятнадцати девять → 8:45, без четверти восемь → 7:45 |
| четверть N+1 | четверть девятого → 8:15 |
| N с половиной | девять с половиной → 9:30 |
| в N часов | неоднозначно → warning + дефолт 09:00 или уточнение на Confirm |

### Относительное время

| Фраза | |
|-------|---|
| через N минут / часов / час | now + delta |
| через полчаса | now + 30 мин |
| через полтора часа | now + 90 мин |
| через пару часов | now + 2 часа |
| через несколько минут | now + 5 мин |
| через неделю | +7 дней (дефолт утро) |

### Абсолютная дата

- `dd.MM`, `dd.MM.yyyy`
- `25 мая`, `25 мая 2026` (также номинатив: `25 май`, `25 июнь`)
- `9 числа`, `5 числа` (цифра + «числа»)
- Год: если дата в прошлом относительно сегодня — **следующий год**

### Часовой пояс

Всегда **локальный timezone** устройства (`ZoneId.systemDefault()`). В UI подтверждения показывать смещение при необходимости.

## Score system (candidate engine)

| Тип | Score | Примечание |
|-----|-------|------------|
| Дата `dd.MM.yyyy` | 100 | самая точная |
| Дата `25 мая` | 90 | |
| «Следующий» день недели | 85 | |
| Порядковый день (`девятого числа`, `9 числа`) | 85 | |
| День недели | 80 | |
| Выходные (`на выходных`) | 75 | |
| Послезавтра | 70 | |
| Завтра | 60 | |
| Сегодня | 50 | |
| `через неделю` | 45 | |
| `через N дней` | 40 | |
| Время `HH:mm` | 100 | |
| Время `HHmm` | 95 | |
| `в N часов M минут` | 90 | |
| половина / без четверти / четверть | 90 | |
| `в N вечера/утра` | 80 | |
| «утром/вечером/днём/ночью в N» | 85 | |
| Полночь/полдень | 75 | |
| `N с половиной` | 85 | |
| `в N часов` | 60 | ambiguous |
| `в N` (short) | 50 | ambiguous |
| `утром`/`вечером` | 40 | fallback |

## Confidence (эвристика)

| Условие | confidence |
|---------|------------|
| Найдены и дата, и время явно (HH:mm или «завтра в 9:00») | 0.92 |
| Только относительный день + «утром» | 0.75 |
| Только «через час» / «через полчаса» | 0.85 |
| «в девять» без am/pm и без «утра/вечера» | 0.5 + warning |
| Время не найдено | 0 + NO_TIME_FOUND |

При `confidence < 0.7` на ConfirmScreen показывать предупреждение жёлтым.

---

## Прошедшее время

Если распознанный `fireAt` < now:

- Если в фразе было **«сегодня»** и время утром уже прошло → предложить **завтра** то же время (warning `PAST_TIME_ADJUSTED`).
- Иначе не сохранять автоматически — пользователь выбирает на Confirm.

## Опциональный облачный fallback (фаза 5)

Запрос только если:

- `confidence < 0.5` и пользователь включил «Улучшенный разбор» в настройках;
- есть `OPENROUTER_API_KEY`.

Ответ API — строгий JSON:

```json
{
  "fireAtIso": "2026-05-18T09:00:00+03:00",
  "body": "позвонить соседу",
  "confidence": 0.95
}
```

Перед постановкой alarm всё равно **ConfirmScreen**.
