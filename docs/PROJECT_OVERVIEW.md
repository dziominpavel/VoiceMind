# VoiceMind — обзор продукта

> Дата: 2026-05-29 · Статус: **MVP реализован** (фазы 0–3). Фазы 4+ — в плане.

## Идея

**VoiceMind** — голосовая напоминалка. Пользователь **говорит** задачу с указанием времени; приложение **понимает, когда напомнить и о чём**, просит подтвердить и **гарантированно будит** в нужный момент выбранным способом.

Пример: *«Завтра в девять ноль-ноль позвонить соседу»* → срабатывание **завтра 09:00**, текст: **«Позвонить соседу»**.

Философия:

- **Быстрее, чем заводить будильник вручную** — один голосовой ввод.
- **Доверие к времени** — всегда экран подтверждения; не ставить alarm без явного согласия пользователя.
- **Offline-first для ядра** — распознавание и базовый парсинг без сети; сеть только как улучшение для сложных фраз.

## Аудитория

- Пользователи Android, которым удобнее сказать напоминание, чем ковырять поля даты/времени.
- MVP: **русский** язык фраз и интерфейса.

## Основные сценарии

### S1. Создать напоминание голосом

1. Главный экран → кнопка микрофона.
2. Пользователь говорит фразу с временем.
3. STT → текст целиком.
4. Парсер → `fireAt` (Instant/epoch) + `body` (текст напоминания).
5. **Экран подтверждения:** дата/время (редактируемые), текст, режим оповещения (по умолчанию из настроек).
6. «Сохранить» → Room + `AlarmManager`.

### S2. Создать вручную (запасной путь)

1. Кнопка «Ввести текстом» или иконка клавиатуры.
2. Те же шаги парсинга и подтверждения (парсер работает и для набранного текста).

### S3. Парсер не уверен / не нашёл время

1. Подсветить проблемное поле.
2. Предложить **ручной выбор даты и времени** (DatePicker + TimePicker).
3. Текст напоминания оставить из распознанной фразы или пустым.

### S4. Список напоминаний

- Вкладка **«Предстоящие»**: сортировка по времени срабатывания ↑ (ближайшие сверху).
- Вкладка **«История»**: уже сработавшие / отменённые / просроченные без действия.

### S5. Срабатывание

1. В `fireAt` срабатывает alarm.
2. `ReminderDelivery` по настройкам напоминания или глобальному default:
   - обычное уведомление (звук + heads-up),
   - «как будильник» (громче, full-screen intent, повтор),
   - только вибрация,
   - тихое уведомление (без звука),
   - (позже) только всплывающий текст.
3. Действия: **Выполнено**, **Отложить** (10 мин), **Отменить**.

### S6. Настройки

- Режим оповещения **по умолчанию** для новых напоминаний.
- (Фаза 4+) Тихие часы — в этот интервал принудительно «тихий» режим.
- (Фаза 4+) Запрос исключения из оптимизации батареи с объяснением.
- Язык распознавания: ru-RU.

## Модель данных

### Reminder

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | Long PK | |
| `clientId` | String UUID | Для бэкапа |
| `fireAt` | Long | Epoch millis, локальная TZ устройства |
| `body` | String | Текст напоминания («Позвонить соседу») |
| `rawPhrase` | String? | Исходная фраза после STT |
| `deliveryMode` | Enum | См. NOTIFICATION_MODES.md |
| `status` | Enum | SCHEDULED, FIRED, DISMISSED, CANCELLED, SNOOZED |
| `createdAt` | Long | |
| `snoozeCount` | Int | Сколько раз откладывали |
| `alarmRequestCode` | Int | Стабильный id для PendingIntent |

### AppSettings (DataStore)

| Ключ | Описание |
|------|----------|
| `defaultDeliveryMode` | Режим по умолчанию |
| `confirmBeforeSchedule` | bool, default true |
| `keepRawAudio` | bool, default false (MVP не хранить аудио) |

Аудиофайлы **не обязательны** в MVP: достаточно текста STT. Хранение записи — опция позже.

## Режимы оповещения (кратко)

Подробно: [NOTIFICATION_MODES.md](NOTIFICATION_MODES.md).

| Режим | Поведение |
|-------|-----------|
| `NOTIFICATION` | Стандартный пуш, звук канала |
| `ALARM` | Высокий приоритет, отдельный канал (full-screen — фаза 4) |
| `VIBRATE_ONLY` | Без звука, вибрация |
| `SILENT` | Только в шторке |
| (будущее) `LOUD_ALARM` | Как системный будильник, отдельная Activity |

## Предложения: что добавить / убрать

### Реализовано (фазы 0–3)

| Идея | Зачем |
|------|--------|
| **Экран подтверждения** | confirmBeforeSchedule; парсер → preview → save |
| **Ручной fallback** Date/Time picker | При неудачном разборе или ручном создании |
| **Snooze на уведомлении** | 10 мин; перепланирует alarm |
| **Reschedule после reboot** | `BOOT_COMPLETED` + `MY_PACKAGE_REPLACED` |
| **Exact alarm permission** | Проверка `canScheduleExactAlarms()`, fallback на `setAndAllowWhileIdle` |
| **Ближайшее напоминание на главном** | Карточка next reminder + относительное время |

### Рекомендуем добавить (высокая ценность)

| Идея | Зачем |
|------|--------|
| **Виджет / ярлык «Сказать напоминание»** | После стабильного MVP |
| **Поиск по body / rawPhrase** | Фаза 5 |

### Имеет смысл, но позже

| Идея | Фаза |
|------|------|
| Повторяющиеся («каждый понедельник в 9») | 6+ |
| Тихие часы | 4 |
| Облачный парсер для редких фраз | 5 (fallback) |
| Категории (звонок / встреча / дом) — цвет в списке | 4+ |
| Экспорт/импорт JSON | 5 |
| Геонапоминания | вне scope |

### Рекомендуем **не** делать в MVP (убрать из первой версии)

| Было в черновике «дневника» | Почему убрать |
|----------------------------|----------------|
| Журнал с воспроизведением аудио | Не цель продукта |
| Теги, избранное, архив записей | Заменено статусами напоминания |
| AI-резюме текста | Не нужно для напоминалки |
| Долгое хранение `.m4a` | Достаточно `rawPhrase`; аудио опционально позже |
| Поиск по транскрипту дневника | Достаточно поиска по `body` в списке (фаза 3) |

## Вне scope

- Совместные списки, аккаунты, облачная синхронизация (первая версия).
- Умный дом, SMS, звонки от имени приложения.
- Медицинские дозировки и критичные напоминания жизни (disclaimer в «О приложении»).

---

## Нефункциональные требования

| Требование | Цель |
|------------|------|
| Точность срабатывания | `setExactAndAllowWhileIdle` + проверка после reboot |
| Задержка STT | < 3 с для фразы до 15 с на mid-range |
| Min SDK | **26** (Android 8.0+) — шире, чем GymProgress (29); `targetSdk` 36 для магазина |
| Приватность | Фразы только локально; в логах release не печатать `body` |
